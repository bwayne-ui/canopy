# Retrospectives

One file per workflow run. Generated by the report-writer subagent at the end of each `monthly-close`, `quarterly-reporting`, or `investor-onboarding` workflow.

## Template

```markdown
# Retrospective — <workflow> — <date> — <entity>

## What went well
-

## What didn't
-

## Surprises
-

## Action items
- [ ]
```

Retrospectives are read by the eval runner to spot recurring issues that should become regression tests or skill refinements.

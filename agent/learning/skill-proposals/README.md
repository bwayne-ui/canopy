# Skill Proposals

Drafts of new skills (or refinements to existing skills) generated automatically by the eval runner or by the `/propose-skill` slash command.

**Nothing here is live.** Promotion to `agent/skills/` requires human review and dual approval (governance change-control rule §9).

## Review checklist
- [ ] Does the proposal cite specific past sessions or audit-log entries?
- [ ] Is there an eval case that fails without this skill and passes with it?
- [ ] Does it duplicate an existing skill?
- [ ] Does it conflict with any rule in `agent/memory/feedback/`?
- [ ] Is the prompt clear, concrete, and citation-driven?
